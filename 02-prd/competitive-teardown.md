# Competitive Teardown — 竞品实地巡检说明

> **前置步骤：** 本文档应嵌入 product-definition-prd SKILL 的 Step 1，或在 01-research 交接后、02-PRD 起草前独立执行。
> **教训来源：** ILAW Lesson Plan Generator 项目（2026-06）—— keyword-research 阶段的竞品分析只看了竞品首页，漏掉了 11 个 SEO 内容页、Budget of Work 下载页、GCash 本地支付、Push 通知营销等关键信息，导致 PRD 页面矩阵和定价决策不全。

---

## 一、为什么要做竞品实地巡检

Keyword-research 阶段的竞品分析目标是 **"验证关键词机会 + SERP 缝隙"** ，它只回答"这个赛道能不能进"。它不做、也不应该做竞品功能拆解。

但 PRD 需要知道的是：

> **竞品到底做了哪些功能？哪些页面？怎么收钱？内容策略是什么？哪些做得比我们好？**

这组问题必须通过 **实地巡检** 来回答——打开竞品每个页面，实际操作一次，像用户一样走完完整流程。

---

## 二、ILAW 项目实测教训（具体漏了什么）

| 发现层面 | 竞品实际做了什么 | 我们 PRD 阶段怎么写的 | 实际影响 |
|----------|----------------|---------------------|---------|
| **内容策略** | 11 个学科/年级博客页 + Budget of Work 下载页 | 页面矩阵 12 页，没有 BoW 页，没有这么多学科专页 | 上线后需要补 SEO 内容 |
| **支付方式** | GCash 扫码 + 手动激活 token | 只考虑了 PayPal（$1.99/月） | 菲律宾老师主要用 GCash，PayPal 门槛高 |
| **PPT 输出** | 10+ 张可上课教学幻灯片（目标/词汇/活动/测验/总结/作业） | 把 PPT 当成表格换格式 | 输出价值低于竞品 |
| **营销手段** | 顶部通知栏用 Tagalog Push "明天周日了还没教案？" | 没有任何用户留存/召回设计 | 用户用完就走，无二次触达 |

**每一条如果能在 PRD 阶段发现，产品和定价决策都会不同。**

## 三、竞品实地巡检 Checklist

### Step 1：打开竞品首页

- [ ] 截图/记录首页完整布局（导航、CTA、表单、页脚）
- [ ] 记录所有导航链接 → 这是竞品内容策略的直接输出
- [ ] 记录页面底部 footer 链接 → 可能有隐藏的 SEO 内容页

### Step 2：走完一次完整用户流程

- [ ] 新用户如何开始？（需注册？免费试用？）
- [ ] 填写表单到生成结果的全过程
- [ ] 输出格式有哪些（Word / PPT / PDF / 在线预览）
- [ ] 输出质量如何？直接可以用还是要改？
- [ ] 下载是否需要扣 token / 付费？

### Step 3：分析付费与定价

- [ ] 免费额度是多少？
- [ ] 付费方式（信用卡 / PayPal / GCash / 银行转账）
- [ ] 定价结构（订阅 / token / 一次性 / 手动转账）
- [ ] 付费流程是否自动化？还是需要人工激活？

### Step 4：枚举所有页面

打开竞品所有可访问的页面（从导航 + footer + sitemap + robots.txt 找）：

```
├── 首页
├── 登录/注册
├── 定价页（如果有）
├── 工具页/生成页
├── 博客/内容页（逐页记录 URL 和 H1）
├── 政策页（Privacy / Terms）
├── 帮助页 / FAQ
├── 特殊资源页（Download / Template / Budget of Work 等）
```

- [ ] 记录每个页面的 URL、页面类型、目标关键词（从 H1 和 URL 推断）
- [ ] 竞品 SEO 做得如何？（Title / Description / Schema / 内链）

### Step 5：总结输出到 `02-prd/competitive-teardown.md`

输出格式（建议）：

```markdown
# Competitive Teardown: [竞品名称]

## 基本信息
- URL
- 定位/一句话
- 技术栈推测

## 内容策略（页面清单）
| 页面 | URL | 类型 | 目标关键词（推断） |
|------|-----|------|--------------------|
| ...  | ... | ...  | ...                |

## 功能清单
- [功能1]
- [功能2]
- ...

## 定价/支付
- 免费额度：
- 付费方式：
- 价格：
- 激活方式：

## 对比我们的差异化机会
| 竞品什么比我们好 | 我们什么比竞品好 |
|-----------------|-----------------|
| ...             | ...             |

## 建议
- P0 必须跟进：
- P1 值得做：
- 不做：
```

## 四、什么时候做

- **必做场景：** 直接竞品有可用产品（能注册、能生成、能付费）
- **不必做：** 竞品只是内容站/博客（没有工具产品）
- **时机：** 01-research 确认"🟢 BUILD_NOW" 之后、02-PRD 起草之前

---

> **原则：不要用 keyword-research 的竞品扫描代替实地巡检。前者回答"能不能做"，后者回答"做什么、怎么做"。**
